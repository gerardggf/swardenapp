# Contribuir a Swarden

Gràcies per voler contribuir a Swarden! Aquest document explica com reportar problemes, proposar millores i enviar canvis.

## Com puc ajudar?

- Reporta bugs obrint una Issue amb passos de reproducció, comportament esperat i entorn.
- Proposa funcionalitats descrivint el cas d’ús i l’impacte.
- Revisa PRs i participa a discussions del roadmap i al Project board públic.

## Flux de treball de desenvolupament

1. Fes un fork del repositori i crea una branca des de main:
   - `feature/<breu-descriptiu>` per a funcionalitats
   - `fix/<breu-descriptiu>` per a correccions
2. Fes canvis petits i enfocats; afegeix o actualitza proves quan calgui.
3. Executa lint i tests locals assegurant que tot passa.
4. Escriu commits clars. Recomanació: https://www.conventionalcommits.org:
5. Actualitza la documentació rellevant (README, docs/).
6. Obre un Pull Request explicant el “què” i el “per què”, agrega captures si cal.

## Estàndards de codi

- Segueix els linters i formatejadors definits al repositori.
- Mantén la compatibilitat i no trenquis APIs públiques sense justificació i notes de migració.
- Nomenclatura, comentaris i proves coherents amb el codi existent.


## Llicència de contribucions

Al enviar un PR, acceptes que la teva contribució es publiqui sota la llicència del projecte (Apache-2.0). No aportis contingut que no puguis llicenciar.



Gràcies per fer Swarden millor!
